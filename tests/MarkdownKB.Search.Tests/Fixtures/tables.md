# 技術選型比較

本文件整理 Phase 2 各元件的技術選型評估結果，供團隊參考。

## 向量資料庫比較

以下是常見向量資料庫方案的比較：

| 方案 | 部署方式 | 與 PostgreSQL 整合 | 授權 | 備註 |
|------|---------|-------------------|------|------|
| pgvector | PostgreSQL extension | 原生 | Apache 2.0 | 已選用，維運成本低 |
| Pinecone | SaaS | 需額外 API | 商業 | 無需自建，但有雲端依賴 |
| Weaviate | 自建 / SaaS | 需額外服務 | BSD 3 | 功能豐富，架構複雜 |
| Qdrant | 自建 / SaaS | 需額外服務 | Apache 2.0 | Rust 實作，效能佳 |
| Chroma | 自建 | 需額外服務 | Apache 2.0 | 適合開發測試 |

**結論：** 選用 pgvector，與現有 PostgreSQL 共用同一資料庫，降低維運複雜度。

## Embedding 模型比較

| 模型 | 維度 | 定價（每 1M tokens） | 中文支援 | 備註 |
|------|------|---------------------|---------|------|
| text-embedding-3-small | 1536 | $0.02 | 良好 | 已選用，性價比最高 |
| text-embedding-3-large | 3072 | $0.13 | 優秀 | 精度更高，成本 6.5x |
| text-embedding-ada-002 | 1536 | $0.10 | 良好 | 舊版，不建議新專案使用 |

**結論：** 選用 `text-embedding-3-small`，足夠支援中文語意搜尋需求，每月預估費用低於 $5。

## 搜尋 API 框架比較

| 方案 | 語言 | 整合難度 | 維護成本 | 備註 |
|------|------|---------|---------|------|
| ASP.NET Core Web API | C# | 低（同語言） | 低 | 已選用 |
| FastAPI | Python | 中（跨語言） | 中 | 適合 ML 重型應用 |
| Express.js | TypeScript | 中（跨語言） | 中 | 生態系豐富 |

**結論：** 搜尋 API 整合於現有 ASP.NET Core 應用，避免引入額外技術棧。
